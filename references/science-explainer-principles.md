# Science Explainer Principles

这份文档沉淀的是可迁移的科普原理，不是某个题材或某个 UP 的表面模板。`video-maker` 做任何科普题材时都应先用这些原则检查内容与画面。

## Core Thesis

好科普不是把知识点讲一遍，而是让观众获得一个新的可用模型。

每一期都必须回答三件事：

- 观众原来为什么会误解？
- 这个主题背后真正决定结果的变量、结构或机制是什么？
- 看完后，观众下次遇到相似问题时能多判断什么？

如果答案只是“知道了一个名词”或“看了一堆资料”，这一期还没有完成。

## Problem-driven Prompting

问题驱动提示词能显著增加深度，因为它强迫内容先定义“要解决的认知卡点”，再决定资料、结构和画面。

默认内容提示词：

```text
不要先解释主题。先找出这期真正要解决的问题：
1. 观众现在最可能相信的简单答案是什么？
2. 这个答案为什么看起来合理？
3. 它在哪个条件、尺度、证据或边界上失效？
4. 如果只能改变观众一个判断，要改变什么？
5. 需要哪种证据载体才能让这个改变成立？
6. 最小满意答案长什么样：机制、证明、对比、规则、流程，还是边界？
7. 哪些问题虽然有趣，但会分散主线，必须延期？
```

`problem_contract` 是 outline 之前的入口。没有问题链，就不要开始排知识点。

问题链应该形成：

- `root_question`: 全片唯一主问题。
- `false_easy_answer`: 观众或普通内容最容易给出的浅答案。
- `difficulty_source`: 这个问题深在哪里。
- `sub_questions`: 2-5 个推进主问题的子问题。
- `answer_shape`: 这一期最后应该交付哪种答案。
- `minimum_satisfying_answer`: 少于这个就没有真正回答问题。

## Principle 1: Start From A False Intuition

开头不要从定义开始。先找到观众自然会带进去的错误直觉。

可用入口：

- 看起来违反常识的现象。
- 一个大家都以为懂、其实解释错的问题。
- 一个真实世界里的新变化、事故、争议、实验、产品或发现。
- 一个简单对象背后不简单的机制。

`opening_contract` 必须写清：

- `false_intuition`
- `why_it_feels_plausible`
- `where_it_breaks`
- `opening_question`
- `promised_answer`

## Principle 2: Compress Complexity Into One Model

科普不是把复杂性消灭，而是把复杂性压进一个观众能拿住的模型。

常见模型：

- `system boundary`: 哪些东西在系统内，哪些是外部条件。
- `flow`: 物质、能量、信息、力、资金、注意力或因果如何流动。
- `constraint`: 真正限制结果的瓶颈是什么。
- `tradeoff`: 为什么不能同时全要。
- `invariant`: 哪个量在变化中保持不变。
- `scale jump`: 微观机制如何变成宏观结果。

每个 beat 都应服务其中一个模型，不服务模型的细节进入 `deferred_details`。

## Principle 3: Use Evidence Carriers

可信感来自证据载体，不来自语气自信。

证据载体可以是：

- 公式、推导、坐标系、变量映射。
- 真实实验、测量、道具、对照组。
- 论文、官方资料、网页、数据表、新闻原文。
- 真实截图、原始界面、日志、测试记录。
- 实拍过程、产线、器材、档案、地图。
- 可追溯的结构图、关系图、时间线。

每个核心 claim 都要进入 `evidence_map`。如果没有证据载体，就要明确它只是类比、假设、示意或推测。

每期默认先选：

- `primary_evidence_carrier`
- `supporting_evidence_carrier`
- `proof_anchor_beats`

常见对应关系：

- 数学/物理：公式、坐标系、板书推导、动画变量、边界条件。
- 自然/生物：实物、栖息地、行为、标本、物种鉴定依据、尺度参照。
- 工程/硬件：实物拆解、测试条件、数据图表、热/力/信号/材料路径、失效点。
- 医学/生命科学：论文、实验设计、样本边界、机制图、风险/收益区间。
- 地理/历史/社会科学：地图、档案、时间线、因果链、原始材料截图。
- AI/软件/工具：真实界面、日志、输入输出样例、流程图、失败案例、能力边界。

证据载体不是越多越好；一主一辅通常比满屏来源更清楚。

## Principle 4: Make The Visual Do Cognitive Work

画面必须承担解释责任，而不是只负责好看。

每一段至少有一个认知工作者：

- `orient`: 让观众知道现在在系统哪一层。
- `compare`: 让差异一眼可见。
- `explain`: 展示变量、机制、流程、因果或结构关系。
- `prove`: 展示证据载体。
- `hedge`: 标出边界、不确定性或示意性质。

如果画面只是氛围图、壁纸、泛素材、漂亮背景，就不能进入终版，除非它只出现在极短的情绪过渡中。

每个 `shot_intent` 应声明：

- `visual_job`: `orient` / `compare` / `explain` / `prove` / `hedge`
- `evidence_carrier`
- `model_element`: system boundary / flow / constraint / tradeoff / invariant / scale jump
- `viewer_should_notice`

## Principle 5: Make Imagegen Accurate By QA

AI 视觉可以直接负责审美、空间感、材质、剖面感、概念图、短中文、公式、标签、数字、来源和 UI 状态；事实可靠性靠目标文本/来源记录 + 视觉核对 + 不合格重生。

默认流程：

- `imagegen_direct_render`: imagegen 直接生成主图、短文字、公式、标签、数字和关键状态。
- `source_targets`: 在内容文件中保留目标文字、公式、数据、来源和边界，供 QA 对照。
- `motion_shell`: Remotion 只做分层揭示、流程推进、状态切换、变量变化、字幕和时间轴。
- `visual_qa`: 逐字逐项看图；不准确就重生。

规则：

- 文字、数字、公式、单位、组件名、来源和关键标签默认交给 imagegen 直接生成，但必须视觉核对。
- 不知道的内部结构不要画成确定事实。
- 示意图必须标注 `示意`、`非按比例`、`概念结构` 或 `为便于观察已夸张`。
- 精美和诚实同时成立时才算通过。

`imagegen` 的正确任务是提高视觉清晰度和审美密度，不是替代证据。它特别适合生成难拍、难看清或需要统一风格的画面；但画面里的真值必须有来源目标，并通过视觉 QA。

## Principle 6: Show Scale Jumps Deliberately

科普经常失败在尺度跳跃太随意：刚讲微观结构，下一秒就跳到社会影响，中间没有桥。

常见尺度链：

- 变量 -> 机制 -> 现象 -> 判断。
- 部件 -> 系统 -> 任务 -> 代价。
- 个体案例 -> 统计规律 -> 边界条件。
- 公式项 -> 几何直觉 -> 物理意义。
- 原料 -> 工序 -> 质量控制 -> 规模化。

每一次尺度跃迁都应有一句口播桥和一个视觉桥。

## Principle 7: Keep The Explanation Minimal, Not Empty

极简不是少做，而是只留下能增加理解的东西。

默认预算：

- 一屏一个主判断。
- 一个主视觉中心。
- 一个支持层。
- 一个临时解释层。
- 不超过三个 annotation clusters。
- 同屏中文层级不超过两级，来源/证明小字可作为第三层。

删掉一半装饰后，如果解释更清楚，应该保留删减版。

## Principle 8: Use Rhythm As Reasoning

节奏不是剪辑花样，而是推理速度。

经验范围：

- 5-12 秒给一个小结论或可见变化。
- 15-30 秒给一个问题、转折或尺度跃迁。
- 高密度解释每 2-6 秒改变视觉层级。
- 慢推导、实验观察、证据展示可以放慢，但必须让观众知道为什么慢。

字幕不应完整复制旁白。字幕应承担：

- 结论锁定。
- 术语锁定。
- 变量标记。
- 反直觉提示。
- 章节转折。

## Principle 9: End With A Transferable Tool

结尾不要只复述“所以答案是……”。要把模型还给观众。

结尾应至少给出一个：

- 判断规则。
- 观察清单。
- 边界条件。
- 常见误判提醒。
- 可迁移类比。
- 下一次看同类问题时的提问方式。

好的结尾让观众感觉自己变聪明了一点，而不是只被喂完了一段资料。

## Principle 10: Use Human Anchors Carefully

人的故事、生活场景、产品体验和社会后果能让抽象知识落地，但不能替代机制。

人类锚点的任务：

- 说明为什么现在值得看。
- 帮观众感受代价或收益。
- 让变量进入真实情境。
- 给抽象结论一个可记住的入口。

不要让情绪故事把证据链挤掉。

## Acceptance Questions

每期验收时问：

- 开头是否抓住了一个真实误解，而不是制造假悬念？
- 观众能否说出这期的唯一核心模型？
- 每个核心 claim 是否有证据载体或边界声明？
- 画面是否真的解释了关系、变量、机制或证据？
- AI 生成视觉和真实事实层是否清楚分离？
- 是否存在没有桥的尺度跳跃？
- 极简审美是否让理解更轻，而不是让信息变薄？
- 结尾是否给了可迁移判断工具？
